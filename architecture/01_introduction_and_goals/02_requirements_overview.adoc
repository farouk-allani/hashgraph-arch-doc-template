== Functional Requirements

[cols="1,8", id=requirements, options="header"]
|===
|ID | Requirement

|[[RE1]]R1 | Automatic Hedera wallet creation upon client registration with secure private key storage
|[[RE2]]R2 | Daily extraction of client activity data from Middle DB (J-1 refresh cycle)
|[[RE3]]R3 | Rule-based points calculation engine processing Middle DB data
|[[RE4]]R4 | Automatic minting of HTS loyalty tokens based on calculated points
|[[RE5]]R5 | Gift redemption workflow with Admin approval and token burning
|[[RE6]]R6 | Super Admin panel for Admin lifecycle management
|[[RE7]]R7 | JWT-based authentication with refresh token rotation
|[[RE8]]R8 | Regular reconciliation between database and on-chain token balances
|[[RE9]]R9 | Storage of historical point calculations and redemption logs
|[[RE10]]R10 | Web and Mobile application interfaces for clients and administrators
